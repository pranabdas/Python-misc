:::tip Good to know

Due to how the numbers are stored in computer memory, floating point algebra
sometimes might produce unexpected results. Notice the following discrepancy
(floating point addition/ subtraction is not associative):

```python
>>> (1.2 + 1E20) - 1E20
0.0
>>> 1.2 + (1E20 - 1E20)
1.2
```

:::

### Loosely typed
As we have seen, we do not need to define the datatype in python. Interestingly,
for certain operations, we can even mix types:
<Tabs
  defaultValue="input"
  values={[
    { label: 'Input', value: 'input', },
    { label: 'Output', value: 'output', },
  ]
}>

<TabItem value="input">

```python
my_str = "Rain! "

```

</TabItem>

<TabItem value="output">

```python
'Rain! Rain! Rain! '
```

</TabItem>
</Tabs>
